Use Case Description

Use Case No

UC-07

Use Case

Recommend Crops

Goal

To provide crop recommendations to farmers based on their location, soil type, and weather conditions.

Preconditions

Farmer must be logged in. The system has access to the farmer's location data.

Success End Condition

Farmer receives a list of suitable crops.

Failed End Condition

The system is unable to provide recommendations.

Primary Actors

Farmer

Secondary Actors

System

Trigger

Farmer requests crop recommendations.

Main Success Flow

Step

Action

1

Farmer navigates to the "Recommend Crops" page.

2

The system automatically fetches the farmer's location.

3

Farmer provides soil type information.

4

Farmer submits the request.

5

The system analyzes the data and provides a list of recommended crops with details.

Alternative Flows

Step

Branching Action

2a

Location not found: The system prompts the farmer to enter their location manually.

3a

Farmer skips providing soil type. The system uses general soil data for the location.

5a

No suitable crops found: The system informs the farmer that no suitable crops were found for the given criteria.

Quality Requirements

Step

Requirement

1

The recommendations should be generated within 5 seconds.

2

The crop data used for recommendations should be up-to-date (not older than 6 months).

3

The system should provide at least three crop recommendations if possible.

4

The user interface should be intuitive and easy to use, even for users with limited technical skills.
