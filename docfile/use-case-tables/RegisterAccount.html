Use Case Description

Use Case No

UC-01

Use Case

Register Account

Goal

Allows a new user (Farmer or Agricultural Expert) to create an account in the system.

Preconditions

User must have access to the registration page.

Success End Condition

User has a new account and is logged into the system.

Failed End Condition

User account is not created, and the user is not logged in.

Primary Actors

Farmer, Agricultural Expert

Secondary Actors

System

Trigger

User navigates to the registration page and submits the registration form.

Main Success Flows

Step

Action

1

User selects their role (Farmer or Agricultural Expert).

2

User enters their personal information (name, email, password, etc.).

3

User submits the registration form.

4

System validates the provided information.

5

System creates a new user account.

6

System authenticates the user and redirects to their dashboard.

Alternative Flows

Step

Branching Action

3a

User provides invalid or incomplete information.

3a1

System displays an error message. User corrects the information and resubmits the form.

4a

The provided email address already exists in the system.

4a1

System displays an error message indicating the email is already in use. The user is prompted to either log in or use a different email address.

Quality Requirements

Step

Requirement

1

The registration process should be completed within 60 seconds.

The system must provide clear feedback on validation errors.
