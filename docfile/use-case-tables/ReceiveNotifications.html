Use Case Description

Use Case No

UC-16

Use Case

Receive Notifications

Goal

To inform users about important events and updates.

Preconditions

User is logged into the system.

Success End Condition

User receives and views the notification.

Failed End Condition

User does not receive the notification.

Primary Actors

User (Farmer, Consultant)

Secondary Actors

System

Trigger

An event occurs in the system (e.g., new message, weather alert).

Main Success Flow

Step

Action

1

The system generates a notification.

2

The system sends the notification to the user's device.

3

The user opens the notification to view the details.

Alternative Flows

Step

Branching Action

3a

User dismisses the notification: The notification is removed from the notification center.

Quality Requirements

Step

Requirement

1

Notifications should be timely and relevant.

2

Users should be able to manage their notification preferences.
