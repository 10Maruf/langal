Use Case Description

Use Case No

UC-17

Use Case

Collect Field Data

Goal

To allow data operators to collect and input agricultural field data into the system.

Preconditions

Data Operator is logged in and has the necessary permissions.

Success End Condition

Field data is successfully collected and stored in the system.

Failed End Condition

Field data is not collected or is stored incorrectly.

Primary Actors

Data Operator

Secondary Actors

System

Trigger

Data Operator initiates the data collection process.

Main Success Flow

Step

Action

1

Data Operator navigates to the "Collect Field Data" page.

2

Data Operator selects the type of data to be collected (e.g., soil samples, crop yields).

3

Data Operator enters the data into the system via a form.

4

The system validates and saves the data.

Alternative Flows

Step

Branching Action

3a

Invalid Data Entry: If the data entered is invalid, the system provides an error message. Data Operator corrects the data and resubmits.

Quality Requirements

Step

Requirement

1

The data collection form should be user-friendly and efficient.

2

The system must ensure data integrity and accuracy.
