Use Case Description

Use Case No

UC-10

Use Case

Forecast Weather

Goal

To provide farmers with weather forecasts to help them plan agricultural activities.

Preconditions

The user has an active internet connection.

Success End Condition

The weather forecast for the specified location is displayed to the farmer.

Failed End Condition

The weather forecast is not displayed.

Primary Actors

Farmer

Secondary Actors

System, External Weather API

Trigger

Farmer requests a weather forecast.

Main Success Flow

Step

Action

1

Farmer navigates to the weather forecast section of the application.

2

Farmer enters a location or uses their default location.

3

Farmer clicks the "Get Forecast" button.

4

The system sends a request to the external weather API with the specified location.

5

The system receives the weather data and displays it to the farmer, including temperature, precipitation, wind speed, and humidity.

Alternative Flows

Step

Branching Action

2a

Invalid Location: If the farmer enters an unrecognized location, the system displays an error message and prompts for a valid location.

4a

API Failure: If the weather API is unavailable, the system displays an error message and suggests trying again later.

2b

No Location: If no default location is set and none is entered, the system prompts the user to specify a location.

Quality Requirements

Step

Requirement

1

Performance: The weather forecast should be displayed within 5 seconds of the request.

2

Accuracy: The weather data should be sourced from a reliable and accurate weather service.

3

Usability: The forecast should be presented in a clear, easy-to-understand format with visual icons.

4

Reliability: The system should handle API failures gracefully and provide informative feedback to the user.
